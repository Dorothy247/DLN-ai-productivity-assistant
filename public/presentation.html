Core Features04 / 10
Architecture

Simple, fast, client-first

🎨 Front-end

Single-file HTML/CSS/JS. Dark glassmorphism UI. Live color-palette studio powered by CSS variables.

🧠 AI Layer

Direct fetch() to Anthropic's /v1/messages using Claude Sonnet 4.6.

🛡️ Fallback

Graceful placeholder() if the API fails — the app never breaks.

User → DLN AI UI → fetch(api.anthropic.com/v1/messages) ↓ (on failure) placeholder() → friendly guidance card
Architecture05 / 10
Prompt Engineering

Every feature is a carefully designed prompt

Role → Context → Task → Constraints → Format. Consistent, testable, refinable.

You are an executive communications assistant for a learning organization. Draft a {tone} email to a {audience} about {topic}. Requirements: - 120–180 words - Clear subject line - One call-to-action - Never invent facts; ask for missing details - Output as: Subject: … \n\n Body: …

Role

Anchors tone & expertise

Constraints

Prevents hallucination and length drift

Format

Makes output easy to render in UI

Prompt Engineering06 / 10
Responsible AI

Built with guardrails, not around them

✅ What we do

  • Show "AI-generated — please review" disclaimers
  • Never store user data server-side
  • Prompts instruct the model to ask when unsure
  • Human-in-the-loop review before sending

⚠️ Known limits

  • Model may reflect training biases
  • Not a source of truth for facts/policy
  • API downtime → placeholder fallback
  • Users control tone, audience, final copy
